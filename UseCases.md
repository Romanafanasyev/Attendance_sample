# Web-приложение I2 study для студентов.
## 1. Список акторов:

- **Студенты** могут:
- [ ] Регистрироваться
- [ ] Авторизовываться
- [ ] Смотреть расписание
- [ ] Смотреть баллы
- [ ] Смотреть свой профиль
- [ ] Смотреть список присутствующих с ними на занятии
- [ ] Отмечаться на занятии
- [ ] Выйти из аккаунта
- **Преподаватели** могут:
- [ ] Авторизовываться
- [ ] Смотреть расписание
- [ ] Генерировать код для отметки на уроке
- [ ] Смотреть свой профиль
- [ ] Смотреть список присутствующих на их занятии
- [ ] Выйти из аккаунта

## 3. Спецификации прецедентов использования:
| Прецедент использования | Регистрация |
| :------ | :--- |
| Краткое описание | Пользователь регистрируется в web-приложении и получает доступ к страницам с его информацией  |
| Действующие лица | Студент |
| Предусловия | Пользователь хочет авторизоваться в веб-приложении, но у него нет аккаунта |
| Основной поток | Пользователь вводит имя пользователя и пароль. Приложение отправляет http запрос, и сохраняет аккаунт пользователя в базе данных.|
| Альтернативные потоки | Пользователь ввел имя пользователя, которое уже есть в базе данных, выводится сообщение о том, что такой пользователь уже зарегистрирован |
| Постусловия | Если прецедент использования завершился успешно, пользователь авторизовывается, и получает доступ к различным страницам web-приложения |

| Прецедент использования | Авторизация |
| :------ | :--- |
| Краткое описание | Пользователь авторизуется в web-приложении и получает доступ к страницам с его информацией  |
| Действующие лица | Студент/Преподаватель |
| Предусловия | Пользователь хочет авторизоваться в веб-приложении |
| Основной поток | Пользователь решает авторизоваться в web-приложении, если у него нет аккаунта, регистрируется, если аккаунт есть, то входит в него. Аккаунты преподавателей предварительно заносит в базу данных администратор. |
| Альтернативные потоки | Пользователь ввел неправильный пароль при входе в аккаунт, выводится сообщение об ошибке |
| Постусловия | Если прецедент использования завершился успешно, пользователь получает доступ к различным страницам web-приложения |
  
| Прецедент использования | Просмотр расписания |
| :------ | :--- |
| Краткое описание | Этот прецедент использования позволяет пользователю увидеть его расписание на неделю |
| Действующие лица | Студент/Преподаватель |
| Предусловия | Пользователь хочет узнать свое расписание на неделю |
| Основной поток | Пользователь нажимает на кнопку "Расписание", переходит на страницу с расписанием, web-приложение с помощью http запроса получает расписание из базы данных. (Расписания должны быть предварительно внесены в базу данных администратором), пользователь смотрит расписание|
| Альтернативные потоки | В базе данных нет уроков, тогда на странице отображается пустое расписание |
| Постусловия | Если прецедент использования завершился успешно, пользователь может посмотреть расписание |
 
| Прецедент использования | Просмотр баллов |
| :------ | :--- |
| Краткое описание | Этот прецедент использования позволяет пользователю увидеть его баллы за экзамены и посещаемость |
| Действующие лица | Студент |
| Предусловия | Пользователь хочет узнать свои баллы |
| Основной поток | Пользователь нажимает на кнопку "Баллы", переходит на страницу с баллами, web-приложение с помощью http запроса получает баллы студента из базы данных. (Баллы периодически заносит в базу данных администратор), пользователь смотрит свои баллы |
| Альтернативные потоки | В базе данных нет баллов, тогда на странице отображается нулевые баллы |
| Постусловия | Если прецедент использования завершился успешно, пользователь может посмотреть свои баллы |

| Прецедент использования | Генерация кода для начала урока |
| :------ | :--- |
| Краткое описание | Этот прецедент использования позволяет пользователю сгенерировать код для отметки посещаемости студентов |
| Действующие лица | Преподаватель |
| Предусловия | Пользователь хочет начать урок и сгенерировать код |
| Основной поток | Пользователь нажимает на кнопку "START LESSON", выбирает урок, который хочет начать, web-приложение генерирует код, и с помощью http запроса заносит его в базу данных. |
| Альтернативные потоки | В базе данных нет уроков на сегодня, тогда пользователь не сможет выбрать его |
| Постусловия | Выводится сгенерированный код |

| Прецедент использования | Отметка присутствия |
| :------ | :--- |
| Краткое описание | Этот прецедент использования позволяет пользователю отметить свое присутствие на занятии |
| Действующие лица | Студент |
| Предусловия | Пользователь хочет отметиться на занятии |
| Основной поток | Пользователь нажимает на кнопку "START LESSON", выбирает урок, на котором хочет отметиться, вводит код, который сообщил ему учитель. web-приложение с помощью http запроса проверяет введенный код |
| Альтернативные потоки | Студент ввел неправильный код, выводится сообщение об ошибке, запись о присутствии не появляется |
| Постусловия | В базе данных появляется запись о присутствии студента |

| Прецедент использования | Просмотр присутствующих на занятии |
| :------ | :--- |
| Краткое описание | Этот прецедент использования позволяет пользователю узнать, кто присутствует на занятии, на котором присутствует он |
| Действующие лица | Преподаватель/Студент |
| Предусловия | Пользователь хочет узнать, кто присутствует на занятии |
| Основной поток | Пользователь нажимает на кнопку "Присутствующие", web-приложение с помощью http запроса выводит список людей, отметивших свое присутствие на уроке, который ведет данный преподаватель |
| Альтернативные потоки | На эту страницу заходит студент, тогда web-приложение выводит список людей, отметивших свое присутствие на том же уроке, на котором отметился данный пользователь |
| Постусловия | Если прецедент использования завершился успешно, пользователь может посмотреть список присутствующих на занятии |

| Прецедент использования | Просмотр профиля |
| :------ | :--- |
| Краткое описание | Этот прецедент использования позволяет пользователю посмотреть его профиль |
| Действующие лица | Преподаватель/Студент |
| Предусловия | Пользователь хочет посмотреть свой профиль |
| Основной поток | Пользователь нажимает на кнопку "Личный кабинет", web-приложение с помощью http запроса выводит данные пользователя (Имя, студент он, или преподаватель, если студент, то номер группы. Списки групп заносит в базу данных администратор)|
| Альтернативные потоки | Администратор не занес информацию о группе студента в БД. Тогда эта информация не выводится. |
| Постусловия | Если прецедент использования завершился успешно, пользователь может посмотреть свой профиль|

| Прецедент использования | Выход из аккаунта |
| :------ | :--- |
| Краткое описание | Этот прецедент использования позволяет пользователю выйти из его аккаунта |
| Действующие лица | Преподаватель/Студент |
| Предусловия | Пользователь хочет выйти из аккаунта |
| Основной поток | Пользователь переходит в "Личный кабинет", нажимает на кнопку "Выйти", web-приложение обновляет информацию о том, что пользователь больше не авторизован |
| Постусловия | Пользователю перестают быть доступны некоторые элементы интерфейса (страницы с баллами, расписанием и т.п.) |
